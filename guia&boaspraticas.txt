Para a construção deste projeto, estamos a usar Programação Orientada a Objetos.
    A utilização das classes, permite uma melhor organização e num só local consiguimos dizer para que serve e do que vai tratar.
    No caso da classe Regras_Porto, definimos:
            as restrições quer de compatibilidade quer capacidade, 
            e a simulação dos sucessores,
        Desta forma conseguimos definir as responsabilidades do Porto em questão

Usamos também o conceito de incrementação que foi lecionado na cadeira de programação imperativa.
    Este conceito foi utilizado para atualizar a fila de espera quando um navio que estava nela vai atracar numa zona.
    Desta forma o navio é retirado da lista de espera


Informações:
Dicionário - é uma estrutura de dados, armazena informações complexas
                o armazenamento funciona da forma key: value
                onde a key normalmente é uma string
                value é o valor associado
    no projeto é o responsável por identificar de forma clara o objeto

Tuple - é uma coleção ordenada e indexada de itens.
    no projeto é usado para representar o estado físico do Porto

Imutabilidade - um objeto é considerado imutável quando o seu estado não puder ser alterado após a sua criação
    No projeto este é um tema importante, pois quando trabalhamos com um estado precisamos de garantir que cada nó visitado é fixo.
    Se fosse mutável o seu conteúdo podia ser alterado o que invalidava todo o processo realizado.

Hash e Hashing - a função hash pega um objeto de entrada, dados, e cria um número inteiro único e fixo
    apenas o que é imutável podem ser transformados num valor hash

    no projeto permite verificar se o estado já foi visitado, logo serve como ferramenta de eficiência


Algoritmo de Busca de Custo Uniforme:
    Resolve problemas de otimização em problemas cujo o objetivo é encontrar o caminho mais barato, ou seja o de menor custo.

    Funciona através da expansão do nó na fronteira que tem o menor custo acumulado, desde o estado inicial até ao momento
        Portanto todas as ligações são importantes.
        Fila de prioridades

    Aqui a profundidade é ignorada e é priorizado o caminho de menor custo total percorrido até ao momento

Algoritmo de Busca em Largura
     Este algoritmo ignora o custo

     E a principal diferença é a forma como lida com a "Fronteira"
        A mesma é vista apenas como uma lista, que vai usar a estratégia de FIFO
